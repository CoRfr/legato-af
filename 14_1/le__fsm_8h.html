Enumeration Type Documentation

enum le_fsm_Policy_t

Defines the error management policy associated to a given FSM. The error concerns unexpected events that may occur for the current state of an FSM.

Enumerator:
LE_FSM_POLICY_IGNORE 

Unexpected event is simply ignored and dropped.

LE_FSM_POLICY_ERROR 

Unexpected event will be reported as an error.

LE_FSM_POLICY_TRAP 

Unexpected event will force the FSM to trap.

LE_FSM_POLICY_MAX 

Not applicable.

enum le_fsm_StateType_t

Defines the nature of a state descriptor.

Enumerator:
LE_FSM_STATE_TYPE_STATE 

Descriptor is a state one.

LE_FSM_STATE_TYPE_MACHINE 

Descriptor is a sub FSM.

LE_FSM_STATE_TYPE_MAX 

Not applicable.

le_result_t le_fsm_AttachMachine ( le_fsm_Context_t  Context,
le_fsm_Machine_t *  Machine 
)

Associate a state machine to the given context. Only one state machine can be attached at a time, and the operation is possible only when the state machine is not running. Once the state machine starts, the sequence to follow in order to change the state machine is:

  • reset the FSM.
  • attach the new FSM.
  • start the new FSM.
Parameters:
Context[IN,OUT] FSM context.
Machine[IN] State machine to attach.
Returns:
Error code
  • LE_OK if the operation succeeded.
  • LE_BAD_PARAMETER if at least one parameter or its content is wrong.
  • LE_NOT_POSSIBLE if the state machine is still running

Generate an event. The generated event can be performed from inside or outside the FSM. This makes it possible to self-sustain the FSM. The specified event is recorded, and will be processed later during the state update (le_fsm_UpdateState()).

le_result_t le_fsm_HandleEvent ( le_fsm_Context_t  Context,
le_fsm_EventId_t  EventId 
)

Inject an event to jump to the next state. Also prime and keep the pump up and running if the state processes generate events. Each injected event will crushed the previous one, as only one event can be injected at a time.

Parameters:
Context[IN,OUT] FSM context.
EventId[IN] Injected event.
Returns:
Error code
  • LE_OK if the operation succeeded.
  • LE_BAD_PARAMETER if at least one parameter is wrong.
  • LE_NOT_POSSIBLE if the FSM is started already.
  • LE_OUT_OF_RANGE if the given event can't be handled from the current state, meaning no transition was found.
Note:
Function may trap according to the error policy that has been set during the configuration of the FSM (see le_fsm_Config_t).

le_result_t le_fsm_UpdateState ( le_fsm_Context_t  Context)

Execute the processes associated to the current state of the FSM. A state process can't change the current state of the FSM. Can't be called recursively. The only way to progress within the FSM is to generate an event from an inside state process or outside the FSM.

Parameters:
Context[IN,OUT] FSM context.
Returns:
Error code
  • LE_OK if the operation succeeded.
  • LE_BAD_PARAMETER if at least one parameter is wrong.
  • LE_NOT_POSSIBLE if the state process is in progress.
  • LE_FAULT if the state process is undefined, meaning the state descriptor is corrupted.
