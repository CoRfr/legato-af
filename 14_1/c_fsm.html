This module is used to handle Finite State Machines (FSM).

Definitions

  • a state machine is a set of transitions.
  • a transition is defined by a source state, a target state and an event.
  • a state identifies a step within a sequence of states.
  • an event corresponds to a trigger allowing to switch from a source state to a target state.

Sub FSM

A sub FSM is an FSM called by another FSM. An FSM can be considered a sub FSM when the FSM caller is "called" back by the FSM callee. Each FSM has at least one starting state, named entry point. This means that an FSM can be called by different FSM and from different entry points. For a detailed description of the FSM, see le_fsm_Machine_t.

State pre/post-processing

The purpose of a state is the processes it connects, split into 3 steps:

  • pre-process executed before entering the state.
  • process executed once the state is active.
  • post-process executed before leaving the state. Spliting the process offers the possiblity to factor some parts of the process that may be common between several states.

Error policy

The purpose of the error policy for an FSM, is the ability to detect an unexpected event or transition and to react accordingly. The error policy has 3 levels:

  • LE_FSM_POLICY_IGNORE Leads to drop the event doing nothing else.
  • LE_FSM_POLICY_ERROR Returns an error.
  • LE_FSM_POLICY_TRAP Triggers an exception and return an error. The policy is specified during the configuration of the FSM (see le_fsm_Config_t).

Log

The log feature records changing states and timestamp in a circular buffer. This permits tracking the chain of events. Its main use is for debug purposes. The length of the record is specified when the feature is enabled. The content of trace can be displayed on the default output console. The trace is lost as soon as the feature is disabled.

Statistics

Statisitics are used to record the load of the FSM distributed among its states. The load tracks counter changes associated with the current state. It may help to detect an useless state if this one has a nulled counter.
